# stock-briefing 구조 정리 및 Phase 2-2 통합 보고서

## 1. 최종 결론

프로젝트 구조 정리와 오프라인 정적 QA는 완료했습니다.

- Next.js 코드를 `web/src/`로 통합
- 구현·QA 보고서를 `office-reports/`로 정리
- 공통 QA, Secret 검사, CI 및 배포 문서 추가
- API 모킹 테스트 7/7 통과
- 정적 QA 실패 0건, 경고 3건
- Git push 및 Vercel 배포는 미수행

다만 Phase 2-2는 아직 완료 상태가 아닙니다. 정상적인 npm 의존성 설치와 production build, 실제 Gemini·Supabase 연동, 운영 RLS 적용, 브라우저·모바일 검증이 남아 있습니다. 또한 기준 문서에 명시된 종목 필터 UI가 현재 화면에 구현되지 않았습니다.

## 2. 구조 정리 결과

웹 소스는 다음과 같이 단일 구조로 정리했습니다.

```text
web/
├── src/
│   ├── app/
│   │   └── api/ask/
│   ├── components/
│   └── lib/
│       └── server/
├── tests/
└── package.json
```

주요 변경 사항:

- `web/app`, `web/components`, `web/lib`를 `web/src/` 아래로 이동
- TypeScript 별칭을 `@/* → ./src/*`로 변경
- API 테스트와 문서의 이전 경로 참조 수정
- Phase 2-2 보고서 4개를 `office-reports/`로 통합
- `.DS_Store`, 빈 이전 디렉터리, Python·TypeScript 캐시 정리
- 중복 `web/.gitignore`를 제거하고 루트 `.gitignore`로 통합
- `pipeline/`, `db/`, `web/tests/`, 정적 대시보드 데이터는 기존 위치 유지

`docs/data/`와 `docs/index.html`은 GitHub Pages 운영 산출물이므로 삭제하지 않았습니다.

## 4. 남은 차단 사항

1. `package-lock.json`과 완전한 `node_modules`가 없습니다.
2. `npm run lint`, `npm run typecheck`, `npm run build`를 정상 설치 환경에서 통과해야 합니다.
3. 종목 필터 UI와 `company` 요청 연동이 누락됐습니다.
4. 운영 Supabase에 최신 `db/schema.sql`을 적용해야 합니다.
5. 실제 Gemini·Supabase 정상·빈 결과·오류 시나리오 검증이 필요합니다.
6. 모바일, 다크모드, 접근성 및 production 번들의 Secret 노출 검사가 필요합니다.
7. 공개 API rate limit 정책이 아직 정해지지 않았습니다.

## 5. 권장 후속 순서

```bash
cd web
npm install
npm run lint
npm run typecheck
npm run test:api
npm run build
cd ..
scripts/qa.sh --build
```

이후 다음 순서로 진행합니다.

1. `package-lock.json` 검토 및 CI를 `npm ci`로 전환
2. 종목 필터 UI 구현
3. 운영 Supabase 스키마 적용
4. 실제 Gemini·Supabase 통합 테스트
5. 데스크톱·모바일 브라우저 QA
6. production 번들 Secret 검사
7. 검증 결과 기록 후 Git 반영 및 Vercel 배포
