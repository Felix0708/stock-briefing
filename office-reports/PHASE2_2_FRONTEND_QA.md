또는 Vercel 연결 정보가 없다. 로컬 실행에 필요한 `web/package-lock.json`과 production build 결과

## Preview QA 실행 체크리스트

아래 표는 Preview URL이 생기면 위에서 아래 순서로 실행한다. 상태 유도를 위해 Production
설정은 바꾸지 않으며, 실패 상태는 Preview 전용 배포 또는 브라우저 응답 지연으로만 만든다.

| 항목 | 실행 절차 | 합격 기준 | 증거 |
|---|---|---|---|
| 정상 답변 | 전체 종목 예시 질문 1개 제출 | 답변, DART 출처 1건 이상, HTTPS DART 링크, 질문·버튼 잠금/복구 정상 | URL·시각·응답 200·화면 캡처 |
| 빈 결과 | 존재하지 않는 종목 필터로 질문 후 `필터 지우고 다시 검색` 선택 | 빈 결과 안내와 필터 원인 문구 표시, 재검색 시 필터가 제거됨 | 두 상태 캡처·응답 200 |
| 429 | 같은 IP에서 Gemini를 호출하지 않는 잘못된 JSON POST를 제한 초과까지 전송한 뒤 UI에서 질문 | 429, `Retry-After`, 초 단위 카운트다운, 전송 잠금, 탭 복귀 시 경과시간 보정, 0초 후 재시도 활성 | Network 헤더·시작/복귀/종료 캡처 |
| 503 | 필수 Upstash 값 하나를 뺀 Preview 전용 배포에서 질문 | 503과 고정 한국어 오류, Secret 원문·내부 오류 미노출 | Preview 배포 ID·응답 본문·화면 캡처 |
| 60초 timeout | 브라우저에서 `/api/ask` 응답을 61초 이상 지연 | 60초 전후 요청 중단, timeout 고정 문구, 다시 시도 가능 | 시작/종료 시각·화면 캡처 |
| 사용자 취소 | 지연 응답 상태에서 `요청 취소` 선택 | 즉시 요청 중단, `요청을 취소했습니다.`, 다시 시도 가능 | 취소 전후 캡처 |
| 모바일 | 320·360·375·768px에서 초기·로딩·오류·정상 화면 확인 | 가로 스크롤·잘림·겹침 없음, 40px 지우기 버튼 사용 가능 | 뷰포트별 캡처 |
| 다크모드 | OS 다크모드와 reduced-motion 각각 전환 | 텍스트·테두리·포커스 식별 가능, 불필요한 모션 억제 | 상태별 캡처 |
| 한글 IME | 질문란에 `삼성전자 최근 공시` 조합 중 Enter, 조합 완료 후 Enter, Shift+Enter | 조합 중 오발송 없음, 완료 Enter 1회 전송, Shift+Enter 줄바꿈 | 입력·Network 요청 수 기록 |
| 키보드 | 마우스 없이 Tab·Shift+Tab·Enter·Space로 필터부터 재시도까지 이동 | 논리적 순서, 포커스 표시, 모든 버튼 실행 가능, 결과 카드로 포커스 이동 | 포커스 순서 메모·캡처 |
| 접근성 | 접근성 트리와 200% 확대 확인, 라이트 주요 텍스트 대비 측정 | label/name·role·live region 존재, 200% 기능 손실 없음, 일반 텍스트 대비 4.5:1 이상 | 트리·대비 수치·캡처 |

429 테스트의 잘못된 JSON 요청은 앱 요청 제한만 소비하고 Gemini 호출 전에 종료된다. 503용
Preview는 확인 후 폐기하고 정상 환경변수의 Preview를 다시 배포한다. timeout은 서버 설정을
고치지 않고 브라우저 측 응답 지연으로 검증한다.

## 재개 조건

1. Vercel 프로젝트 연결 및 Preview 배포를 완료해 URL을 기록한다.
2. 인앱 브라우저가 연결된 세션에서 위 표의 전 항목을 다시 실행한다.
3. 각 상태의 실제 응답 코드·화면·포커스·접근성 트리와 뷰포트별 캡처 경로를 이 문서에 기록한다.
4. Preview 전 항목 통과 후 사장님 GO/STOP을 받고, GO일 때만 Production 배포와 스모크 QA를 수행한다.

코드 수정, 커밋, push, 배포는 수행하지 않았다.
