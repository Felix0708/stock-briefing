# 주식 AI 브리핑 서비스 — 기획서

> 포트폴리오 겸 수익화 실험 프로젝트. 월 유지비 0원.

## 1. 한 줄 정의

관심 종목을 등록하면, 매일 아침 해당 기업의 신규 공시(DART)를 AI가 요약해서 이메일로 보내주는 서비스.

## 2. 왜 이 프로젝트인가

- **경력 서사와 일치**: 소프트뱅크에서 구축한 "데이터 수집 파이프라인 → LLM 연계 → 운영 자동화" 구조를 개인 스케일로 처음부터 끝까지 재현. 면접에서 "회사 시스템의 미니어처를 혼자 만들었다"고 말할 수 있음.
- **실사용자 확보 용이**: 첫 사용자가 나 자신(주식 투자자). 만들고 버려지는 포트폴리오가 아님.
- **PdM 지향 서사**: Phase 3에서 결제·지표를 직접 붙여보며 "기술과 비즈니스를 연결"하는 경험 확보.

## 3. 단계별 로드맵

### Phase 1 — 브리핑 MVP (2~3주)
- 관심 종목 목록(watchlist) 기반으로 매일 아침 DART 신규 공시 수집
- Gemini로 공시 내용 요약 (무료 티어)
- HTML 이메일 발송 (Gmail SMTP)
- GitHub Actions로 매일 07:30 KST 자동 실행 (서버 불필요)
- **완료 기준**: 내가 매일 아침 실제로 읽는 메일이 온다

### Phase 2 — 공시 RAG Q&A (4~6주)
- 공시 원문을 Gemini 임베딩으로 벡터화 → Supabase(pgvector) 저장
- Next.js 웹 앱: "이 회사 최근 CB 발행 조건이 뭐야?" → 근거 공시 링크와 함께 답변
- 소프트뱅크 경력(NiFi→Milvus→RAG→Dify)의 개인 버전 완성
- **완료 기준**: 웹에서 질문하면 출처 포함 답변이 나온다

### Phase 3 — 서비스화 + 수익화 실험 (여유 시)
- 로그인(소셜 로그인) + 사용자별 관심 종목 관리
- 보유 종목 입력(수량·평균단가) → 내 포트폴리오 관점의 브리핑/알림
- 웹 푸시 or 카카오톡 알림
- 무료: 3종목 제한 / 유료: 무제한 + Q&A
- 토스페이먼츠 or Stripe 결제 연동, 가입자·리텐션 지표 수집
- **완료 기준**: 남이 가입해서 쓸 수 있는 서비스가 된다

## 4. 아키텍처 (Phase 1)

```
GitHub Actions (매일 07:30 KST, 무료)
   │
   ▼
Python 파이프라인
   ├─ 1. DART OpenAPI: watchlist 종목의 전일~당일 신규 공시 목록 조회
   ├─ 2. 공시 원문 다운로드 → 텍스트 추출
   ├─ 3. Gemini Flash (무료 티어): 종목별 요약 + 투자 관점 코멘트
   └─ 4. Gmail SMTP: HTML 브리핑 메일 발송
```

### 기술 선택 이유

| 선택 | 이유 |
|---|---|
| Python | 이력서 BE 스킬과 일치. Phase 2 임베딩(sentence-transformers) 생태계도 Python |
| GitHub Actions cron | 서버 비용 0원. cron 스케줄 실행이 곧 "인프라 자동화" 어필 포인트 |
| Gmail SMTP | 무료, 앱 비밀번호만 발급하면 끝. 나중에 Resend 등으로 교체 가능 |
| Gemini Flash | 무료 티어 하루 1,500 요청. 하루 수십 건 요약에 충분 |
| 이메일 우선 (웹 없음) | Phase 1에서 프론트를 빼면 2~3주 완성 가능. 제품은 "아침에 읽는 메일" 그 자체 |

## 7. 리스크와 대응

- **Gemini 무료 티어 축소**: 모델명을 환경변수로 분리해 교체 용이하게 설계. 최악의 경우 로컬 LLM(Ollama)로 전환 가능
- **공시 원문이 너무 긴 경우**: 문서당 최대 글자 수 제한 후 요약 (config로 조절)
- **투자 조언 규제**: 요약·정보 제공에 한정하고, 메일 하단에 "투자 권유 아님" 고지 문구 포함
